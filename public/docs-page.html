‚Äî Entanglement. In the quantum world, two qubits located even light-years apart can still act in ways that are strongly correlated. Quantum computing takes advantage of this entanglement to encode problems that exploit interdependence between qubits.

In 1996, Mikl√≥s Ajtai made a significant contribution to Lattice-based cryptography by introducing a cryptographic construction that relied on the hardness of lattice problems. Collaborating with Cynthia Dwork, they demonstrated the difficulty of solving the Short Integer Solutions (SIS) problem, an average-case lattice problem, which proved to be as challenging as worst-case lattice problems.

Two years later, Jeffrey Hoffstein, Jill Pipher, and Joseph H. Silverman introduced a lattice-based public-key encryption scheme called NTRU. Although their scheme did not demonstrate the same level of hardness as solving a worst-case lattice problem, it marked an important milestone in lattice-based cryptography.

In 2005, Oded Regev presented the first lattice-based public-key encryption scheme with a security proof based on worst-case hardness assumptions. Regev's work also introduced the Learning with Errors (LWE) problem. Since then, researchers have dedicated significant efforts to improving the security proofs and efficiency of Regev's original scheme.

The field of lattice-based cryptography has witnessed extensive development, with additional cryptographic primitives being constructed based on LWE and related problems. Notably, Craig Gentry introduced the first fully homomorphic encryption scheme in 2009, which was also based on a lattice problem.

The history of quantum computing and post-quantum cryptography spans several decades, with numerous algorithms developed since 1996. The NIST Post-Quantum Cryptography Competition has been a significant milestone in this journey, aiming to standardize post-quantum cryptographic algorithms. The European Commission has also played a vital role in funding the PQCRYPTO project, which focuses on the development of post-quantum cryptography since 2015.

In the CRYSTALS-Kyber scheme, the security of the key exchange is based on the assumption that solving the LWE problem is computationally difficult. The scheme involves generating public and private keys, and during the key exchange, the parties use these keys to establish a shared secret key securely. PQC "Stateless" Hash-based digital signature

From Lamport to PQC SPHINCS+


1. Lamport signature:

In 1979, Leslie Lamport introduced a groundbreaking method for one-time signatures. Lamport's approach involved the creation of two data sets comprising 256 random 256-bit numbers, referred to as Set A and Set B. Each of these sets formed the private key, consisting of a total of 512 values. By hashing each random number, a corresponding set of 512 hashes was derived, which constituted the public key. This article explores Lamport's method, highlights its limitations, and delves into the subsequent development of Merkle trees as a solution.

Lamport Method:

Lamport's method operates by hashing the message and then examining each bit of the resulting hash (ranging from 0 to 255). If a bit is 0, the corresponding element from Set A is used; otherwise, the corresponding element from Set B is employed. The signature itself comprises 256 random numbers selected from either Set A or Set B, while the public key consists of the 512 generated hashes. Notably, both the public and private keys measure 16KB in size, making Lamport signatures computationally demanding.

Limitations of Lamport Signatures:

Despite its innovative nature, the Lamport method suffers from a fundamental drawback. With this approach, a unique key pair is required for each individual message, which poses scalability and efficiency challenges. Recognizing this limitation, researchers sought alternatives that could overcome the need for a new key pair for every message.


2. The Emergence of Merkle Trees:

In the same year as Lamport's groundbreaking work, a solution to the key pair issue arose through the concept of Merkle trees. Merkle trees, named after Ralph Merkle, are a hashing technique widely used in various applications, including Bitcoin, Ethereum, Git distribution systems, No SQL databases, and P2P networks. By hashing and pairing data elements, Merkle trees construct a hierarchical structure, culminating in a root element known as the root hash. This root hash serves as proof of knowledge of all other hashes in the tree while keeping them concealed.

Utilizing Merkle Trees for Signatures:

The utilization of Merkle trees in the context of signatures presented a significant advancement. Instead of creating a new key pair for each message, Merkle trees allowed for the authentication of multiple messages with a single key pair. This was achieved through the concept of Many-Time Signatures (MTS) within a hypertree. By employing a hierarchical structure, MTS enabled the authentication of a large number of Few-Time Signatures (FTS) key pairs.


3. One time signature:

In the pursuit of improving the performance and reducing key sizes of signature methods, Robert Winternitz proposed the Winternitz One-Time Signature (W-OTS). Named after its creator, W-OTS allows for the simultaneous signing of multiple bits, presenting a significant advancement in the field. This article explores the principles and methodology behind W-OTS, shedding light on its key features and demonstrating its functionality through a visual illustration.

Winternitz OTS Method:

The Winternitz OTS method begins by generating 32 random numbers, each consisting of 256 bits. These 32 values constitute the private key. Subsequently, each of these values undergoes 256 iterations of hashing, resulting in the generation of 32 corresponding values that form the public key.

To create a signature using W-OTS, the message is hashed using the SHA-256 algorithm. This process yields 32 8-bit values, denoted as N1, N2, ..., N32. Each of these 8-bit values is then utilized to determine the number of iterations for subsequent hashing operations. Specifically, the value N determines the number of times the 8-bit signature value will be hashed (256 - N times).

To verify the signature, the original message is hashed using SHA-256 once again. The resulting hash is then processed by examining each 8-bit value. For each value, the 8-bit signature value is hashed a number of times equal to the corresponding message hash value (N1, N2, ...). The outcome of each operation should equal the corresponding value in the public key.


4. HORS:

Hash to Obtain Random Subset (HORS) is an advanced quantum-robust, hashed-based one-time signature scheme. By leveraging the power of hashing and indexing, HORS enables the creation of private and public key pairs to achieve secure and unique signatures. This article explores the methodology behind HORS, highlighting its key steps and demonstrating its application through a simple example.

HORS Methodology:

The HORS method begins by generating a series of integers to serve as private key values. These values, ranging from priv0 to priv255, are organized in an indexable form. Subsequently, each private key value undergoes a hashing process, resulting in a list of corresponding public key values, denoted as pub. These public key values are indexed for future reference.


5. XMSS Hash-based signature schemes:

Such as the Merkle Signature Scheme (MSS), have played a vital role in various applications. However, MSS suffers from performance and memory limitations. To overcome these challenges, enhanced methods like XMSS and SPHINCS have emerged. This article delves into the drawbacks of XMSS and introduces SPHINCS, a stateless variant that addresses the state-related issues while preserving the core methods of XMSS.

Merkle Signature Scheme (MSS):

The core principle behind MSS lies in Merkle trees, which enable efficient hashing and verification of data elements. However, MSS encounters significant performance and memory concerns, limiting its practicality in certain scenarios.

XMSS and its Stateful Nature:

To enhance the efficiency of MSS, the XMSS method was introduced. XMSS reduces the performance overhead associated with MSS but introduces a stateful nature to the signature scheme. This statefulness implies that private keys must be retained and not reused, posing challenges in managing and securing a growing number of private keys.

SPHINCS+:

SPHINCS+ (Stateless" PHotonic Isogeny-based Signature Scheme), a hybrid signature scheme that combines hash-based, code-based, and isogeny-based components to achieve statelessness and post-quantum security.

In a post-quantum world, where powerful quantum computers exist, the need to eliminate or reduce reliance on state in cryptographic systems arises due to the potential impact of quantum computing on traditional cryptographic algorithms.

The idea behind "stateless digital signature" is to eliminate the state that is stored by the signer between signatures, why? because quantum-computers using superposition that means it can in the two probability in multiple states at once. This state can be used by an attacker to forge signatures, so eliminating it makes the signature scheme more secure.

In a stateful digital signature scheme, the signer stores a secret key and a signing state between signatures. The signing state is used to generate the signatures, and it can be used by an attacker to forge signatures if they are able to obtain it.

In a stateless digital signature scheme, the signer does not store any state between signatures. Instead, the signing state is generated on the fly when the signer wants to create a signature. This makes it much more difficult for an attacker to forge signatures, as they would need to obtain the secret key and the random seed that was used to generate the signing state.

Stateless digital signature schemes are more efficient than stateful digital signature schemes, as they do not require the signer to store any state between signatures. They are also more secure, as they are less vulnerable to attacks.

Stateful cryptographic systems typically involve the storage of sensitive information, such as secret keys or internal states, which are used to perform cryptographic operations. These systems can be more vulnerable to attacks in a post-quantum world because quantum computers could potentially exploit the stored information to compromise the security of the system.

By eliminating or reducing state in cryptographic systems, the aim is to develop stateless algorithms that do not rely on the storage of sensitive information or the maintenance of long-term states. Stateless algorithms operate solely based on the input provided at the time of the cryptographic operation without relying on any stored information. This can potentially enhance the security of cryptographic systems against attacks that target the stored information.

Statelessness can reduce the attack surface and mitigate certain types of attacks, such as side-channel attacks that aim to extract sensitive information from the system's state. By eliminating state, cryptographic systems become more resistant to attacks that target stored secrets or exploit vulnerabilities in state management.

SWIFFTX NIST submission in 2008:

SWIFFTX

SWIFFT is a collection of provably secure hash functions. It is based on the concept of the fast Fourier transform (FFT). SWIFFT is not the first hash function based on FFT, but it sets itself apart by providing a mathematical proof of its security. It also uses the LLL basis reduction algorithm. It can be shown that finding collisions in SWIFFT is at least as difficult as finding short vectors in cyclic/ideal lattices in the worst case. By giving a security reduction to the worst-case scenario of a difficult mathematical problem, SWIFFT gives a much stronger security guarantee than most other cryptographic hash functions.

SWIFFT is a family of homomorphic hash functions provided as part of a candidate submission to the NIST hash function competition in 2008 by Vadim Lyubashevsky, Daniele Micciancio, Chris Peikert, and Alon Rosen. Unfortunately NIST was reject in first round in SHA3 competition. The family of SWIFFTX has an interesting set of characteristics:

1. Provably collision-resistant: it is provably computationally hard to find a binary-valued x given its image under f that is chosen randomly.

2. Universal: for any x_1,x_2 in the domain the probability that f(x_1) = f(x_2) over the choice of f in the family is the inverse of the size of the range.

3. Regular: if x is distributed uniformly then f(x) is distributed uniformly for any f in the family.

4. Randomness extractor: even if x is not distributed uniformly, f(x) is distributed uniformly over the choice of f in the family

5. Homomorphic: each f in the family is homomorphic over the domain.

6. Facilitates ZKPoKPs: the resulting hash value can be readily used in ZKPoKPs.

7. Facilitates post-quantum digital signatures: its homomorphism property enables short post-quantum hash-based digital signature schemes, such as K2SN-MSS.

8. Constant-time: any f in the family is free of data-dependent branching and therefore facilitates avoidance of timing side-channel attacks.

SWIFFTX as one of the fastest secure hash functions available, which is collision-resistant and supports zero-knowledge proofs of knowledge of a preimage (ZKPoKP) and post-quantum digital signatures

1. ECC can be broken with a well-known vulnerability:

Most existing blockchain technology is using Elliptic Curve for generated key pair, which is that relies on prime integers. These prime integers that protect our private and public keys.

Given an instance of the Elliptic Curve Discrete Logarithm Problem (ECDLP), we have a fixed generator point P in an elliptic curve over a finite field Fp, and a fixed point Q in the subgroup generated by P. The goal is to find the unique integer m such that Q = [m]P.

Shor's algorithm for breaking ECC involves the use of quantum computing.

1. The algorithm starts by creating two registers of qubits, each with a length of n + 1, and initializing them in the |0‚ü© state.

2. A Hadamard transform (H) is applied to each qubit, resulting in a superposition state.

3. Conditioned on the content of the registers, the algorithm performs operations involving the points P and Q, mapping the superposition state to a new state that includes the corresponding multiples of P and Q.

The third register is discarded, and a quantum Fourier transform (QFT) is applied to each of the two registers.

Finally, the measurement of the first two registers, which hold 2(n + 1) qubits in total, provides data that can be used for classical post-processing to compute the discrete logarithm m.

2. ECDSA can be broken with a well-known vulnerability:

Most existing blockchain technology is using Elliptic Curve for digital signature, which is that relies on prime integers. These prime integers that protect our private and public keys.

1. In ECDSA, there is a known elliptic curve point denoted as G, and a public key P that is calculated as P = sG, where s is a secret value known only to the signer.

2. Shor's algorithm maps the problem of finding the secret value s to a problem in period finding. It does this by defining a function F that takes a pair (a, b) and maps it to the point aG - bP.

3. The goal is to find the period d of the function F, which can be achieved using the principles of quantum superposition and quantum Fourier transform (QFT).

4. Once the period d is determined, it can be used to recover the private key s by performing certain mathematical operations involving modular division. By finding the private key s, an attacker can forge signatures and break ECDSA.

It is important to note that Shor's algorithm and the details of quantum computing involve complex mathematical concepts, including quantum superposition, QFT, and modular arithmetic. These concepts enable the quantum computer to efficiently solve the problem of finding the private key in ECDSA, which would be computationally infeasible for classical computers.
